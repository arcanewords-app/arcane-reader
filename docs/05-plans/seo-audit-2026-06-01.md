# SEO baseline audit — 2026-06-01

**Host audited:** `https://arcane-reader.vercel.app`  
**Agent:** utility SEO (`.cursor/skills/seo/SKILL.md`)

## Summary

| Area                     | Rating        | Notes                                                                                                         |
| ------------------------ | ------------- | ------------------------------------------------------------------------------------------------------------- |
| Technical crawl          | Good          | `robots.txt` and `sitemap.xml` return 200; disallow rules correct                                             |
| Publication SSR          | Good          | `/p/*` returns injected meta, JSON-LD Book, hidden crawler `<main>`                                           |
| Static page SSR (Vercel) | Fixed in repo | Was serving raw `index.html` for `/`, `/catalog`, `/about`; `vercel.json` now routes SEO paths to `api/index` |
| Canonical HTTPS          | Fixed in repo | Publication canonical used `http://`; `getPublicBaseUrl()` now honors `X-Forwarded-Proto`                     |
| GSC / traffic            | Open          | Search Console not registered; limited non-catalog landing content                                            |
| Content                  | Good sample   | Published works have substantive descriptions and author/translator metadata                                  |

## Findings

### Finding: Static public pages served without SSR meta on Vercel

- Severity: Warning
- Evidence: `curl` on `/`, `/catalog`, `/about` returned 3148-byte static `index.html` (same as Vite build); no per-page `<title>` or canonical in HTML. `vercel.json` catch-all `/(.*) → /index.html` ran before Express.
- Impact: Google and social crawlers saw identical default meta for home, catalog, and info pages.
- Fix: Add rewrites for `/`, `/catalog`, `/about`, `/contact`, `/privacy`, `/terms` → `/api/index` (done in `vercel.json`).
- Confidence: Confirmed
- Owner: API

### Finding: Publication canonical URLs used HTTP scheme

- Severity: Warning
- Evidence: `/p/tsikl-neizbezhnosti-novaya-redakutra` HTML contained `link rel="canonical" href="http://arcane-reader.vercel.app/..."`.
- Impact: Mixed signals vs HTTPS sitemap URLs; possible consolidation issues.
- Fix: `getPublicBaseUrl(req)` in `src/server.ts` using `x-forwarded-proto` (done).
- Confidence: Confirmed
- Owner: API

### Finding: Home and catalog shared identical document title

- Severity: Info
- Evidence: `STATIC_PAGE_META` used the same title for `/` and `/catalog` (descriptions already differed).
- Impact: Weak differentiation in SERP for catalog queries.
- Fix: Catalog title → `Каталог переводов — Arcane` (done in `STATIC_PAGE_META`).
- Confidence: Confirmed
- Owner: API

### Finding: Google Search Console not configured

- Severity: Warning
- Evidence: [`seo-search-console.md`](seo-search-console.md) open tasks unchecked.
- Impact: No coverage/indexing diagnostics or sitemap submission in GSC.
- Fix: Human ops — property, verify, submit `{PUBLIC_URL}/sitemap.xml`, sample URL inspection.
- Confidence: Confirmed
- Owner: human-ops

### Finding: Sitemap chapter URLs capped at 100 publications

- Severity: Info
- Evidence: `SITEMAP_CHAPTER_PUBS_LIMIT = 100` in `src/server.ts`; sitemap lists publication landing pages for all pubs (up to 1000).
- Impact: Deep reading URLs for pubs beyond the first 100 are discovery-only via publication page links.
- Fix: Accept trade-off or plan sitemap index + split files if catalog grows (product decision).
- Confidence: Confirmed
- Owner: API / product

### Finding: Limited traffic landing beyond catalog and publications

- Severity: Info
- Evidence: No blog/how-to section; `/about` exists. Entity filters use query params on `/catalog`.
- Impact: Fewer entry keywords; reliance on publication quality and catalog.
- Fix: Strengthen `/about`; optional future `/how-it-works` with real content; author guide for unique descriptions ([`seo-author-content-guide.md`](seo-author-content-guide.md)).
- Confidence: Likely
- Owner: human-ops / UI

### Finding: Publication content quality (sample)

- Severity: Pass
- Evidence: API sample shows unique titles, multi-paragraph descriptions, `authorDisplay`, `translatorDisplay`, covers, slugs.
- Impact: Aligns with Google helpful-content / AI-search guidance.
- Fix: Keep author checklist; monitor empty descriptions on new publishes.
- Confidence: Confirmed
- Owner: human-ops
